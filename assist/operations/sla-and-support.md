# SLA And Support

## Purpose

Enterprise customers need confidence that CODEXSUN can be supported professionally.

This document defines support expectations and service-level thinking.

## Support Levels

### Standard Support

For starter and growing businesses.

Includes:

- Business hours support.
- Issue tracking.
- Basic troubleshooting.
- Documentation support.
- Release notes.

### Priority Support

For high-dependency businesses.

Includes:

- Faster response.
- Integration issue support.
- Data correction guidance.
- Sync issue support.
- Priority bug triage.

### Enterprise Support

For large tenants or dedicated deployments.

Includes:

- Dedicated support process.
- Tenant health monitoring.
- Backup and restore coordination.
- Staging validation.
- Release coordination.
- Dedicated escalation path.

## Incident Severity

### Severity 1

Critical production outage or data risk.

Examples:

- Billing unavailable for many users.
- Tenant data leak risk.
- Accounting data corruption.
- Production database unavailable.

### Severity 2

Major business workflow degraded.

Examples:

- e-Invoice failures.
- POS sync stuck.
- Reports unavailable.
- Important integration failing.

### Severity 3

Limited issue with workaround.

Examples:

- Specific report error.
- UI issue blocking one workflow step.
- Non-critical notification failure.

### Severity 4

Minor issue or question.

Examples:

- Small UI polish.
- Documentation question.
- Enhancement request.

## Support Rules

- All support access must be audited.
- Data correction must be approved and logged.
- Production changes should follow release or hotfix process.
- Customer communication should be clear and non-technical where possible.
- Incidents should have root cause notes and prevention actions.
